# Internationalization (i18n)

## Languages supported

- **Thai (`th`)** — primary language for the venue audience
- **English (`en`)** — for international participants and federation reporting

## Behavior

- The user can switch language with a toggle.
- The change takes effect **immediately**, without a full page reload.
- Default language preference is **not yet decided** — likely Thai with browser-language detection on first visit.

## Implementation plan

Today, **strings are inlined in components in Thai**. Some pieces of copy use English. There is no live i18n layer yet.

The intended approach:

1. Extract strings into dictionary files: `en.json`, `th.json`.
2. Hold the current language in **React Context** (or a small store like Zustand) — sufficient because language affects rendering only, not server fetches.
3. Place the language toggle in the header / navbar where it is visible across all roles.
4. Server-rendered pages either:
   - Read the language from a cookie / header and render in that language, or
   - Render in the default language and let the client swap on hydration

Open question: whether to use a library (`next-intl`, `react-intl`) or roll a thin custom hook. The codebase is small enough that custom-thin is viable, but library benefits (pluralization, ICU message format, type-safe keys) may justify the dependency.

## Strings that must be bilingual

| Surface | Notes |
|---------|-------|
| All UI labels | Buttons, headers, table column titles |
| Activity log `action` field | "ออกใบเหลือง" vs. "Yellow card issued" |
| Status labels | "กำลังดำเนินการอยู่" vs. "Ongoing" |
| Card semantics | Already encoded as symbols (`>`, `~`), so the symbol is language-agnostic; tooltips/labels translate |
| Error / validation messages | Once forms have real validation |
| Report export | Federation may require Thai or English versions, possibly both |

## Strings that stay one language

| Surface | Stays | Reason |
|---------|-------|--------|
| Athlete name | As entered | Personal name; do not translate |
| Affiliation name | As entered | Proper noun |
| Bib | As entered | Identifier |
| URL paths | English | Convention; Thai URLs cause encoding noise |

## Dates and numbers

- Date format: locale-aware (Thai Buddhist calendar?? — TBD)
- Time format: HH:MM:SS, language-independent
- Distance: `distance_km` is stored as a string; presentation may localize "20 km" vs. "20 กม."

## Theme is not localized

The theme is fixed per route prefix (light for admin/public, dark for in-event roles). See [tech-stack.md](tech-stack.md).
